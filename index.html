🚀 Agile Methodologies Cheat Sheet

Quick reference guide for 9 popular Agile frameworks & Comparison

S Scrum

Iterative framework with fixed-length sprints

Key Concepts

  • Sprint cycles (1-4 weeks)
  • Daily standups
  • Sprint planning & retrospectives
  • Product backlog management

Roles

Product Owner, Scrum Master, Development Team

Artifacts

Product Backlog, Sprint Backlog, Increment

Learn More

  • Scrum.org
  • Agile Alliance - Scrum

✅ Pros

Clear structure, regular delivery, good for complex projects

❌ Cons

Can be rigid, requires experienced team, overhead for small projects

KKanban

Visual workflow management system

Key Concepts

  • Visual board (To Do, In Progress, Done)
  • Work-in-progress (WIP) limits
  • Continuous flow
  • Pull-based system

Core Principles

Start with current process, pursue incremental change, respect current roles

Metrics

Lead time, cycle time, throughput, cumulative flow

Learn More

  • Kanban University
  • Atlassian - Kanban

✅ Pros

Flexible, visual, easy to implement, continuous delivery

❌ Cons

Less predictable, can become chaotic, requires discipline

XPExtreme Programming

Engineering-focused agile methodology

Core Practices

  • Pair programming
  • Test-driven development (TDD)
  • Continuous integration
  • Refactoring
  • Simple design

Values

Communication, Simplicity, Feedback, Courage, Respect

Planning

Release planning, iteration planning, user stories

Learn More

  • ExtremeProgramming.org
  • Agile Alliance - XP

✅ Pros

High code quality, rapid feedback, customer involvement

❌ Cons

Requires skilled developers, intensive, customer availability needed

LLean Software Development

Eliminate waste, optimize flow

7 Principles

  • Eliminate waste
  • Amplify learning
  • Decide as late as possible
  • Deliver as fast as possible
  • Empower the team
  • Build integrity in
  • See the whole

Waste Types

Partially done work, extra features, relearning, handoffs, delays, task switching, defects

Learn More

  • Lean Enterprise Institute
  • Agile Alliance - Lean

✅ Pros

Efficiency focus, waste reduction, faster delivery

❌ Cons

Can be too focused on speed, requires cultural change

CCrystal

Family of methodologies based on team size

Crystal Variants

  • Crystal Clear (2-8 people)
  • Crystal Yellow (8-20 people)
  • Crystal Orange (20-40 people)
  • Crystal Red (40-80 people)

Core Properties

Frequent delivery, osmotic communication, reflective improvement, personal safety

Focus Areas

People over processes, communication, regular delivery

Learn More

  • Alistair Cockburn - Crystal
  • Agile Alliance - Crystal

✅ Pros

Flexible, scalable, people-focused

❌ Cons

Less prescriptive, requires experience to implement

FFeature Driven Development

Model-driven, feature-centric approach

5 Processes

  • Develop overall model
  • Build feature list
  • Plan by feature
  • Design by feature
  • Build by feature

Key Roles

Project Manager, Chief Architect, Development Manager, Chief Programmer, Class Owner

Best Practices

Domain object modeling, developing by feature, component/class ownership

Learn More

  • Agile Alliance - FDD
  • FeatureDriven.org (Community)

✅ Pros

Good for large teams, regular delivery, clear progress tracking

❌ Cons

Less customer involvement, requires upfront modeling

DDSDM Atern

Dynamic Systems Development Method

8 Principles

  • Focus on business need
  • Deliver on time
  • Collaborate
  • Never compromise quality
  • Build incrementally
  • Develop iteratively
  • Communicate continuously
  • Demonstrate control

Project Phases

Pre-project, Feasibility, Foundations, Exploration, Engineering, Deployment, Post-project

Learn More

  • Agile Business Consortium - DSDM

✅ Pros

Business-focused, time-boxed, quality emphasis

❌ Cons

Complex framework, requires training, heavyweight

AAgile Unified Process

Simplified version of Rational Unified Process

4 Phases

  • Inception (scope & architecture)
  • Elaboration (detailed requirements)
  • Construction (build system)
  • Transition (deploy to users)

Key Disciplines

Modeling, Implementation, Testing, Deployment, Configuration Management, Project Management

Agile Values

Staff knows what they're doing, simplicity, agility over ceremony

Learn More

  • Scott Ambler - AUP

✅ Pros

Balance of discipline and agility, good documentation

❌ Cons

Can be heavy, requires RUP knowledge

SFSAFe

Scaled Agile Framework for enterprises

Core Competencies

  • Lean-Agile Leadership
  • Team and Technical Agility
  • Agile Product Delivery
  • Enterprise Solution Delivery
  • Lean Portfolio Management
  • Organizational Agility
  • Continuous Learning Culture

Levels

Essential, Large Solution, Portfolio, Full SAFe

Key Roles / Events

Release Train Engineer (RTE), Product Management, System Architect, PI Planning, System Demo, Inspect & Adapt

Learn More

  • Scaled Agile Framework (Official)
  • Atlassian - SAFe

✅ Pros

Structured scaling, aligns strategy & execution, good for large orgs

❌ Cons

Complex, prescriptive, high overhead, potential for "agile-fall"
